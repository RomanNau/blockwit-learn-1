Структуры данных- определенный способ организации/размещения информации, так что бы ее было можно эффективно использовать в дальнейшем.

1)Дерево- такое вертикальное расположение структуры данных, в каждой папке папке есть подпапки, а внутри еще папки.
Пример из жизни: расположение файлов в телефоне, пк, фотокамеры, расположение файлов внутри установленных программ/приложений.

2)Список- динамическая(изменяемая) структура данных, элементы которого содержат в себе данные и указывают на следующий элемент.
Пример из жизни: допустим книга "Максим Горький, Собрание сочинений" открываем ее, нам нужен определенный рассказ. Сначала ищем вкладку "рассказы", а потом внутри нужный рассказ.То есть вкладка "Рассказы" указывает на нужный нам рассказ.

3)Ассоциативный массив- набор пар элементов, один из которых значение, а второй ключ. При чем ключи все разные и не где не повторяются. Любой элемент можно быстро определить по индивидуальному ключу.
Пример из жизни: номера автомобилей, позывные в армии.

4)Очередь- эта такой организации информации, при котором обработка начинается с первого места, а добавление нового элемента становлением на последнее место. Такой способ носит название по-английски:FIFO (“first in — first out”, “первым вошел — первым вышел”).
Пример из жизни: очередь в поликлинике, магазине.

5) Стейк - противоположной очереди вид организации информации, то есть первым обрабатывается последний элемент, а добавление нового так же приходит на последнее место очереди.
Такой принцип работа называется LIFO — "last in — first out" (“последним пришел — первым вышел”).
Принцип жизни: складирование почты за месяц для постояльца отеля.
